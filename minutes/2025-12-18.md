- **Alex:** Conformance and Impact - static API

  - The original intent was that the impact API would feed live impact reporting into the specs. So if models change we would be able to constantly update it, based on the latest science. Great for people who want the latest info but if people want something static for auditing it could be problematic.

  - Decided: Have a static copy of these numbers provided with the statement version of those specs + Keep a live running copy of the number which we constantly update in the repository. We can replace the existing copy when required. One can choose between the sources.

  - Benefits: It provides assurance for a certified copy for regulators AND the latest up to date numbers based on the latest metrics.

  - First Conformance meeting 17th dec 2025: the conformance API will complement the document of conformance model.
